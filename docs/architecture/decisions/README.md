# Architecture Decisions

This directory records Phoenix Architecture Decision Records (ADRs).

ADRs capture decisions that shape Phoenix as an AI Operations Platform. They are durable records of context, options, decisions, and consequences. They should be short enough to read during implementation work and specific enough to prevent repeated re-litigation of settled direction.

## Index

| ADR | Status | Title |
|---|---|---|
| [0001](0001-phoenix-as-ai-operations-platform.md) | Accepted | Phoenix as an AI Operations Platform |

## Process

1. Copy [ADR_TEMPLATE.md](ADR_TEMPLATE.md).
2. Name the file with the next zero-padded number and a short slug.
3. Record context, decision, alternatives, and consequences.
4. Link related issues, PRs, architecture docs, and implementation work.
5. Update this index.

ADRs can be superseded, but should not be silently rewritten after implementation depends on them.
